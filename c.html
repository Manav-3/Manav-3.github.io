DBMS Extended Questions

1. What is Database and DBMS? List out the advantages and disadvantages of DBMS.

Database: A structured collection of data.

DBMS: A system software for creating and managing databases.

  • Advantages: Reduces redundancy, ensures consistency, supports sharing, provides security, backup and recovery.
  • Disadvantages: Costly, complex, requires skilled staff, risk of failure.

2. Differentiate between file processing system and DBMS. Give at least four points.

  • File systems have high redundancy; DBMS reduces it.
  • File systems lack integrity; DBMS ensures it.
  • Sharing is limited in file systems; DBMS enables it.
  • File systems lack security; DBMS enforces it.

3. Explain the different models of DBMS with advantages and disadvantages.

  • Hierarchical Model: Tree structure. Simple but rigid.
  • Network Model: Graph structure. Flexible but complex.
  • Relational Model: Tabular structure. Easy to use but resource-intensive.
  • Object-Oriented Model: Real-world representation. Powerful but less common.

4. What is a relational database? How is it different from other database models?

A relational database stores data in tables using rows and columns. It uses keys to relate tables and supports SQL. It's more flexible and widely used than hierarchical or network models.

5. What is data redundancy? How does DBMS help in reducing it?

Data redundancy is repeated data. DBMS reduces it by centralizing storage and using normalization.

6. Write differentiate between centralized and distributed database systems.

  • Centralized: Data in one location, simpler but risky if failed.
  • Distributed: Data across locations, more reliable but complex.

7. Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

A DBA manages and maintains the database system.

  • Design and implementation
  • Backup and recovery
  • Performance tuning
  • Security and access control

8. Define normalization. Explain 1NF, 2NF and 3NF with suitable examples.

Normalization: Organizing data to reduce redundancy.

  • 1NF: Atomic values only.
  • 2NF: Full dependency on primary key.
  • 3NF: No transitive dependency.

9. Explain the terms: primary key, foreign key, and candidate key with examples.

  • Primary Key: Uniquely identifies a row. E.g., Student_ID.
  • Foreign Key: Refers to primary key in another table. E.g., Course_ID in Enrollment table.
  • Candidate Key: All potential primary keys. E.g., Email, Roll Number.

10. What is SQL? Explain its components and common functions.

SQL: Structured Query Language used to interact with relational databases.

  • DDL: CREATE, ALTER, DROP
  • DML: SELECT, INSERT, UPDATE, DELETE
  • DCL: GRANT, REVOKE
  • TCL: COMMIT, ROLLBACK
  • Functions: COUNT(), SUM(), AVG(), NOW()

12. Define the following terms.

  • a. Data Dictionary: A centralized repository that stores metadata such as definitions, structures, constraints, and relationships for data elements in a database.
  • b. Primary Key: A field or combination of fields that uniquely identifies each record in a table.
  • c. Relationship: An association between two or more tables in a database, usually established through keys.
  • d. Data Manipulation Language (DML): A subset of SQL used for retrieving, inserting, deleting, and updating data in a database.
  • e. Structured Query Language (SQL): A standard programming language used to manage and manipulate relational databases.
  • f. Data Integrity: The accuracy and consistency of data over its lifecycle, ensured through constraints and rules.
  • g. Data Definition Language (DDL): A subset of SQL used to define and modify database structures, such as tables and indexes.
  • h. Data Security: Protecting data from unauthorized access or corruption through authentication, encryption, and access controls.
  • i. Database System: A combination of the database, DBMS software, and associated applications that manage and provide access to data.

Unit 2 – C Programming

1. Working with a Function

  • a. Define Function: A function is a block of code that performs a specific task and can be reused.
  • b. Syntax of Functions:
    return_type function_name(parameter_list) {
      // body of function
    }
  • c. Types of Functions: Library functions (e.g., printf, scanf) and User-defined functions.
  • d. Components of Function:
    • i. Function Prototype
    • ii. Function Call
    • iii. Function Definition
    • iv. Return Type

2. Categories of Function with Example

  • i. Function with return type but no arguments:
    int getValue() { return 10; }
  • ii. Function with return type with arguments:
    int sum(int a, int b) { return a + b; }
  • iii. Function with no return type and no arguments:
    void greet() { printf("Hello "); }
  • iv. Function with no return type with arguments:
    void display(int x) { printf("%d", x); }

3. Storage Classes

  • i. Automatic: Default for local variables.
  • ii. External: Global variables accessible across files using extern.
  • iii. Register: Requests that variable be stored in CPU register.
  • iv. Static: Retains value between function calls.

4. Recursive Function

A recursive function is a function that calls itself.

int factorial(int n) {
  if(n == 0) return 1;
  else return n * factorial(n - 1);
}

5. Passing Array to a Function

Arrays can be passed to a function by specifying the array name in the parameter.

void printArray(int arr[], int size) {
  for(int i = 0; i < size; i++) {
    printf("%d ", arr[i]);
  }
}

Unit 2 – C Programming

1. Working with a Function

  • a. Define Function: A function is a block of code that performs a specific task, can be reused, and improves modularity.
  • b. Syntax of Functions:
    return_type function_name(parameter_list) {
      // function body
    }
  • c. Types of Functions:
    • Library Functions: Built-in functions like printf(), scanf().
    • User-defined Functions: Functions created by programmers to perform specific tasks.
  • d. Components of Function:
    • i. Function Prototype: Declaration of function before main.
      int add(int, int);
    • ii. Function Call: Calling function where needed.
      add(5, 10);
    • iii. Function Definition: Actual body of function.
      int add(int a, int b) { return a + b; }
    • iv. Return Type: Data type of value returned by function, e.g., int, void.

2. Categories of Function with Example

  • i. Function with return type but no arguments:
    int getValue() { return 10; }
  • ii. Function with return type with arguments:
    int sum(int a, int b) { return a + b; }
  • iii. Function with no return type and no arguments:
    void greet() { printf("Hello\n"); }
  • iv. Function with no return type with arguments:
    void display(int x) { printf("%d", x); }

3. Storage Classes

  • i. Automatic (auto): Default for local variables, lifetime is within the block.
  • ii. External (extern): Declared outside all functions, accessible across multiple files.
  • iii. Register: Requests storage in CPU register for faster access (compiler dependent).
  • iv. Static: Retains value between function calls, lifetime is the entire program.

4. Recursive Function

A recursive function is a function that calls itself to solve a smaller instance of the same problem.

int factorial(int n) {
  if(n == 0) return 1;
  else return n * factorial(n - 1);
}

5. Passing Array to a Function

Arrays can be passed to functions by specifying the array name and size in the parameter list.

void printArray(int arr[], int size) {
  for(int i = 0; i < size; i++) {
    printf("%d ", arr[i]);
  }
}
